What This Tool Does

This app detects Ultra-Low Frequency (ULF) geomagnetic pulsations — tiny magnetic field oscillations in the Pc3–Pc5 range (2–50 mHz, i.e. periods of 20 seconds to 8 minutes) — by measuring how coherently those oscillations appear simultaneously across a network of ground magnetometers.

The central output is a coherogram: a 2-D heatmap with time on the horizontal axis and frequency on the vertical axis, coloured by network-averaged magnitude-squared coherence γ². High coherence (bright yellow in the inferno colourmap) at a given time and frequency means multiple station pairs are all seeing the same oscillation at that moment — a strong indicator of a real ULF wave, not local noise.

Pc pulsations are driven by solar-wind–magnetosphere coupling (Pc3–Pc4) and field-line resonances tied to radiation-belt dynamics (Pc4–Pc5). Detecting them in real time or in archived data is useful for space weather monitoring, radiation-belt studies, and magnetospheric sounding.

Quick Start

  1. Choose a Source (USGS is the default — no account needed, US stations only).
  2. Pick a Date / Time. USGS data is available up to yesterday; SuperMAG data has a ~60-day lag.
  3. Select which Stations to include by clicking the pills in the station bar. Active stations are highlighted.
  4. Press ▶ Run Analysis. Real data is fetched and analysed; results appear across five tabs.
  5. Click any cell in the coherogram to inspect the per-pair coherence spectra for that window.
Tip: Stations with no data at the chosen date appear struck-through after the analysis runs. If most stations fail, try an earlier date or a different source.

Data Sources

Sources are divided into two categories. Real independent data (USGS, SuperMAG) provide a genuinely independent measurement per station — coherence reflects actual geophysical structure. Single-source proxy sources (NOAA SWPC, Synthetic) derive all station signals from a single base measurement and will produce artificially high coherence that does not represent real spatial coherence in the magnetosphere. The app labels the toolbar with a ⚠ Modeled — coherence is synthetic warning whenever a proxy source is active.

USGS Geomagnetism (default)

Fetches real 1-minute cadence data from the USGS Geomagnetism Program API. No account required. Coverage is US observatories only — stations outside the US are automatically dimmed and excluded. Data is typically available up to yesterday. Reliable and fast; the best starting point.

SuperMAG Ground Network

Fetches from the SuperMAG global archive (JHU/APL). Provides genuinely independent per-station data for hundreds of ground magnetometers worldwide, enabling coherence analysis across much larger baselines and higher latitudes. Requires a free SuperMAG user ID (enter it in the SuperMAG ID field). Data has a ~60-day processing lag, so today's date will return nothing — the date picker automatically caps to the valid range.

SuperMAG supports both 1-minute and 1-second cadence. Use 1-sec for Pc3 and Pc4 pulsations (>6.7 mHz), since 1-minute data can only resolve up to ~8.3 mHz (the Nyquist limit).

NOAA SWPC Solar Wind (7-day / 1-day)

Fetches DSCOVR solar-wind Bz data from NOAA. All station signals are derived from this single base series using per-station delay, amplitude scaling, and noise — it is a modelled signal, not independent per-station data. Coherence will be artificially high. Useful for testing the pipeline but not for scientific analysis.

Synthetic Substorm

Generates synthetic signals locally in the browser using a seeded random number generator. Injects Pc3, Pc4, and Pc5 tones at known frequencies and times so you can verify the pipeline is detecting the right bands. Always available, instant, reproducible.

Toolbar Parameters

ParameterDescription
Date / Time Start of the data window to fetch. For USGS, time-of-day is ignored and data starts at midnight. For SuperMAG, the time component is used — enter e.g. 14:30 to begin a 30-minute 1-sec window mid-afternoon. The picker automatically enforces source-specific validity limits.
Cadence Sampling interval for SuperMAG only. 1-min fetches 6 hours of data and resolves frequencies up to ~8.3 mHz (Pc5 and low-Pc4). 1-sec fetches ~30 minutes of data and resolves up to 500 mHz, capturing the full Pc3–Pc5 range. Hidden for other sources.
SuperMAG ID Your free SuperMAG logon ID. Register at supermag.jhuapl.edu. Only required for the SuperMAG source.
Slice Length of each analysis window in samples. The hint (e.g. "30 min") shows the equivalent real time given the current cadence. Longer slices give better frequency resolution but fewer time points. At 1-min cadence, 30 samples = 30 minutes; at 1-sec cadence, 600 samples = 600 seconds = 10 minutes. The number of Welch sub-segments (and thus statistical power) also depends on this.
Step Hop size between successive windows in samples. Smaller steps give finer temporal resolution in the coherogram but increase computation time. The hint shows the equivalent real time. Steps smaller than Slice/2 produce overlapping windows.
Max freq Upper frequency limit displayed on the coherogram y-axis (mHz). The actual resolvable maximum is the Nyquist frequency (500 mHz at 1-sec, 8.3 mHz at 1-min). Setting Max below Nyquist zooms the coherogram into the ULF band of interest.
Component Geomagnetic field component to analyse. H/N (horizontal north) is most sensitive to Pc4–Pc5 field-line resonances and is the standard choice for ULF studies. D/E (declination / east) detects azimuthal wave modes. Z (vertical) is sensitive to ionospheric and compressional modes. SuperMAG uses N/E/Z; USGS uses H/D/Z — the selector maps automatically.
Significance mask When enabled, pixels whose coherence falls below the analytic p<0.05 significance threshold are blacked out. The threshold (γ²crit) depends on the number of Welch segments K: higher K → lower threshold → more sensitive. Shown as a triangle marker on the colourbar and reported in the Sig. Threshold metric card.

Station Selection

Click any station pill in the station bar to toggle it on or off. Active stations are colour-highlighted; unavailable stations (wrong source or missing IAGA code) are dimmed and non-interactive. After an analysis run, stations that returned no data at the chosen date are shown struck-through.

You can add any station by typing its IAGA code (e.g. LER for Lerwick) into the "Add IAGA…" box. If the station's coordinates are not in the built-in list, you can supply them as CODE lat lon Name (e.g. LER 60.1 -1.2 Lerwick) so it appears correctly on the globe.

The active station set and all toolbar settings are saved to the URL hash and to localStorage automatically, so your configuration survives page refreshes and can be shared by copying the URL.

Results Tabs

🔥 Coherogram

The main output. Horizontal axis is time (window centres); vertical axis is frequency (mHz); colour is network-mean γ² using the inferno colourmap (black → purple → orange → yellow). Dashed horizontal lines mark the Pc5|Pc4 boundary (6.7 mHz) and Pc4|Pc3 boundary (22 mHz).

Two display modes are available via the γ² / φ° buttons: coherence magnitude (default) and circular-mean phase angle (useful for measuring propagation delays). Click any cell to open a detail panel showing per-pair coherence spectra for that window.

📈 Time Series

Raw (first-differenced) signal traces for each active station, stacked vertically. Useful for spotting data gaps, spikes, or obvious pulsation bursts by eye before trusting the coherogram.

〜 FFT Spectra

Power spectral density (Welch estimate) for each station over the full dataset. Peaks in PSD at the same frequency across multiple stations give additional confirmation that coherence is driven by a real signal, not an artefact in a single station.

⊞ Correlation Matrix

Mean γ² between every pair of active stations, computed over the full time series. Off-diagonal cells also show the great-circle distance in km between the two stations. Coherence typically decreases with station separation, so this matrix lets you assess spatial coherence scale lengths — a key observable for field-line resonance studies. Click any cell to open the per-pair coherence spectrum detail.

🌍 Station Globe

Orthographic globe view showing station positions as coloured dots and the top-6 station pairs as arcs coloured by coherence (inferno scale). The globe auto-centres on the station network centroid. Drag to rotate to inspect the network from any angle; click ⟲ Reset view to return to the centroid. Stations on the far hemisphere are hidden (correct orthographic clipping).

Note: 1-minute cadence data (Nyquist 8.3 mHz) can only resolve Pc5 and the very bottom of Pc4. Switch to 1-second cadence (SuperMAG) to study Pc3 and Pc4.

Signal-Processing Pipeline

All DSP is implemented from scratch in plain JavaScript (no library). The pipeline runs entirely in your browser.

1
Fetch per-station signals The chosen source is queried for each active station independently. The true sampling rate fs is derived from the median inter-sample interval, not assumed. Stations that return no data are dropped and the remaining signals are trimmed to a common length.
2
First-difference Each signal is first-differenced (x[n] − x[n−1]) to high-pass filter the data and emphasise pulsation-band variability relative to slow baseline drift.
3
Sliding-window Welch coherence A window of Slice samples slides across the time series in steps of Step samples. For each window, every station's Welch FFT segments are computed once and cached, then every station pair's magnitude-squared coherence is computed from the cached segments: γ²[k] = |Pxy[k]|² / (Pxx[k] · Pyy[k]). The network average across all pairs gives one column of the coherogram.
4
Significance threshold The analytic p<0.05 threshold γ²crit = 1 − 0.051/(K−1) is derived from the number of Welch sub-segments K. Higher K → lower threshold → more sensitive detection, but the frequency resolution coarsens (each sub-segment is shorter).
5
Event detection Contiguous time windows whose band-mean coherence exceeds an adaptive threshold (the maximum of the significance threshold, mean+1.5σ, and 0.35) are flagged as events and classified into Pc3/Pc4/Pc5 by their peak frequency.

Tips & Limitations

Getting meaningful coherence

  • Use at least 3–4 stations. With only 2 stations, a single noisy pair dominates the network average.
  • Mix high-latitude and mid-latitude stations to sample both Pc5 resonance nodes and Pc3 activity simultaneously.
  • A Slice of 20–30 minutes (1-min cadence) balances frequency resolution (~1–2 mHz bins) against temporal detail.
  • If coherence is suspiciously high everywhere (>0.9), check that you are not using the SWPC source — it produces modelled, correlated signals.

Interpreting the colourmap

  • The significance mask (p<0.05) is a useful noise filter but conservative — real pulsations near threshold can be masked. Turn it off to see the full picture.
  • Isolated bright pixels are usually noise. Real ULF events appear as coherent bright patches spanning multiple time and frequency bins.
  • The Pc5|Pc4 and Pc4|Pc3 dashed gridlines on the coherogram help identify which band an event falls in.

Data gaps & quality

  • SuperMAG 1-second data is not available for all stations — many observatories only archive 1-minute data.
  • Large data gaps cause spectral leakage. The first-difference step reduces but does not eliminate this effect.
  • USGS data can have occasional NaN / fill-value samples, which are handled by the parser but may shorten the usable record.

Export

  • PNG exports the current coherogram canvas as an image.
  • CSV exports the full coherogram data matrix (rows = frequency bins, columns = time windows) for offline analysis.

Auto-refresh

The ⟳ Auto-refresh button is available for live NOAA SWPC sources and re-runs the analysis every 60 seconds to track the evolving solar-wind coherence structure in near real time. The countdown is shown in the status bar. The button is hidden for historical sources (USGS, SuperMAG) where repeating the same query would return identical data.

Technical Notes

  • Zero build step — pure HTML + JavaScript, runs directly from the filesystem or a static server.
  • The only external dependency is Chart.js 4.4.1 (CDN), used only for the Time Series, FFT, and window-detail line charts.
  • The FFT is an in-place iterative Cooley–Tukey radix-2 algorithm. All FFT lengths are padded to the next power of two.
  • CORS: USGS and SuperMAG APIs do not allow direct browser requests. The bundled proxy.py serves static files on port 8080 and forwards API requests server-side. Run it with python3 proxy.py.
  • Settings (stations, source, date, parameters) are persisted to both localStorage and the URL hash so configurations are bookmarkable and shareable.
